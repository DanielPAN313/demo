# 24h 全天交互 + 排期模块

**Agent Builders Hackathon 南京站 · 赛道二创意智能体方案与明日开发执行稿**

## 一句话定位

我们不是做一个更会提醒人的 AI 日历，而是做一个知道什么时候不该打扰你的 24 小时个人节律 Agent。

---

## 1. 赛道贴合度判断

**结论：贴合赛道二，而且比赛道一更适合。**

赛道二强调跳出行业框架、从个人兴趣和奇思出发，做任意天马行空的 AI 智能体。这个项目如果被表达成“AI 日历”，会显得普通；如果表达成“守护注意力和个人节律的 24 小时智能体”，就更符合创意赛道。

核心差异不是“自动排任务”，而是“判断该不该打扰用户”。

系统同时管理：

- 时间
- 状态
- 任务优先级
- 外部事件
- 触达边界

评委容易记住的三个概念：

- **注意力防火墙**
- **状态感知排期**
- **夜间静默晨报**

## 3. MVP 功能范围

### 必须完成

1. **今日节律首页**
   用一天时间轴展示专注、碎片、休息、睡眠四类时段。

2. **状态感知重排**
   输入“睡眠差 / 压力高 / 紧急任务插入”，自动生成新计划。

3. **注意力防火墙**
   把通知分成立即打扰、稍后提醒、晨报汇总、静默归档。

4. **可解释排期**
   每次调整都给出理由，例如：“睡眠质量低，上午深度任务减少 30%。”

5. **Agent 对话入口**
   用户说“我现在很累，帮我重排下午”，系统返回新排期和解释。

### 可以模拟，不必真实接入

- 睡眠评分、压力等级、疲劳状态可以用按钮或滑块模拟。
- GitHub 紧急 Issue 可以用一条预置事件模拟。
- 课表、任务列表、协作消息可以使用本地 JSON 或写死数据。
- 通知推送不需要系统级推送，只要在界面里展示分类结果。

## 6. 推荐 Demo 剧本

1. 展示今日节律首页：上午是专注时段，下午有任务，晚上是低打扰休息时段。
2. 点击“昨晚睡眠差”：系统自动降低上午任务强度，插入恢复窗口。
3. 点击“GitHub 插入 P1 Issue”：系统判断任务紧急程度和当前状态。
4. Agent 给出两个方案：
   - 现在 25 分钟最小修复
   - 推迟到 14:00，并后移低优先级任务
5. 展示注意力防火墙：
   - 紧急事项立即提示
   - 普通消息延后
   - 夜间消息进入静默归档
6. 切到晨报：展示夜间摘要、今日排期、睡眠状态导致的调整原因。

### 演示冲击点

评委需要看到一个清晰瞬间：

> 用户状态变差 + 外部紧急任务插入时，Agent 没有粗暴提醒，而是判断打扰等级、自动重排，并解释原因。

## 7. 评委眼前一亮的亮点表达

| 亮点 | 怎么讲 |
| --- | --- |
| 注意力防火墙 | 不是提醒器，而是打扰管理器。系统决定什么时候提醒、什么时候延迟、什么时候静默。 |
| 状态感知排期 | 不是按空闲时间排任务，而是按用户的真实精力排任务。 |
| 夜间静默晨报 | 夜间 24h 后台吸收外部变化，但不打扰用户，早晨一次交付摘要。 |
| 可解释排期 | 每次重排都有原因，降低用户对 Agent 黑箱决策的不信任。 |
| 自主权旋钮 | 普通调整自动执行，重大调整先询问，睡眠和课程等锚点不可侵犯。 |
| 个人节律画像 | 系统越用越懂用户什么时候最适合学习、沟通、复盘或休息。 |

## 8. 路演话术

### 开场

现在很多 AI 日历都在帮人把时间排满，但真实用户的问题不是没有提醒，而是被提醒、消息和任务切碎。我们的项目想解决的是：AI 不仅要知道该做什么，还要知道什么时候不该打扰你。

### 方案

我们做的是一个 24 小时状态感知的时间调度 Agent。它读取课表、任务、外部平台变化和用户状态，动态判断现在该做什么、是否要重排、是否应该通知用户。

### 差异

它不是普通 AI 日历，而是注意力防火墙 + 状态感知排期 + 夜间静默晨报。白天主动推进，休息时低打扰，夜间静默吸收，晨起一次交付。

### 收束

我们希望这个 Agent 不只是管理任务，而是学习人的节奏，让用户在该专注时更专注，该休息时真的能休息。

## 9. 最终建议

**明天的产品策略：坚决砍功能，保留一个强故事闭环。**

把完整产品愿景留给路演，把现场开发集中在“状态变化触发智能重排”这个核心瞬间。

- 主标题建议：**24h Personal Rhythm Agent**
- 中文副标题：**一个知道什么时候不该打扰你的时间调度智能体**
- 现场展示优先级：**界面完整 > 重排闭环 > 解释清楚 > 真实接入**
- 失败风险最高的部分是范围过大，必须避免临场加功能。

---

## 10. 最终一句话

我们做的不是 AI 日历，而是一个 **24 小时状态感知的时间调度 Agent**。它根据日程、任务优先级、外部变更和用户身心状态，动态决定“现在该做什么、该不该打扰、是否需要重排”，让 AI 从提醒工具变成真正懂节奏、懂边界、懂状态的时间管家。
